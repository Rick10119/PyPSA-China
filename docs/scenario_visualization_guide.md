# Scenario Result Visualization Guide

This guide explains how to use the `plot_scenario_comparison.py` script to visualize the scenario results generated by `generate_value_test_configs`.

## Overview

`plot_scenario_comparison.py` analyzes the results of the 72 scenario configurations produced by `generate_value_test_configs` and generates:

1. **A 9-subplot comparison chart**: each subplot corresponds to one market–demand combination (3 × 3), showing cost or capacity changes across flexibility scenarios.
2. **A summary table**: key indicators for all scenarios.
3. **Detailed data files**: for further analysis.

## Script Features

### Highlights

- **Config-driven**: reads the base version number from `config.yaml` and constructs the full scenario matrix.
- **Automatic scenario construction**: builds all demand–market combinations (3 × 3 = 9).
- **Fault-tolerant data loading**: displays a notice when data for a scenario is missing.
- **Multi-dimensional comparison**: supports both `costs` and `capacities` file types.
- **Smart categorization**: classifies costs by technology type and cost type.
- **Visualization output**: produces clear comparison charts and tables.
- **Data completeness statistics**: provides detailed missing-data reports.

### Scenario Dimensions

The script handles the following dimensions:

- **Demand**: L (Low), M (Mid), H (High)
- **Market**: L (Low), M (Mid), H (High)
- **Config type**: `100p` (100 % capacity), `non_flexible` (baseline group)

The script automatically constructs 3 × 3 = 9 scenario combinations, each comparing the `100p` and `non_flexible` configurations.

## Usage

### Basic usage

```bash
# Analyze cost changes (default)
python scripts/plot_scenario_comparison.py

# Analyze capacity changes
python scripts/plot_scenario_comparison.py --file-type capacities

# Specify results directory
python scripts/plot_scenario_comparison.py --results-dir /path/to/results

# Specify output directory
python scripts/plot_scenario_comparison.py --output /path/to/output

# Specify config file
python scripts/plot_scenario_comparison.py --config /path/to/config.yaml

# Verbose output
python scripts/plot_scenario_comparison.py --verbose
```

### Arguments

| Argument | Default | Description |
|----------|---------|-------------|
| `--results-dir` | `results` | Path to the results directory containing all `version-*` folders |
| `--output` | `results/scenario_analysis` | Output directory path |
| `--file-type` | `costs` | File type to analyze: `costs` or `capacities` |
| `--config` | `config.yaml` | Config file path (used to obtain the base version number) |
| `--verbose` | False | Enable verbose output |

## Output

### 1. Scenario Comparison Chart

A large figure with 9 subplots, one per market–demand combination:

- **Rows**: Market level (Low, Mid, High)
- **Columns**: Demand level (Low, Mid, High)
- **Content**: cost/capacity change of `100p` relative to `non_flexible`

Each subplot contains:
- Horizontal bar chart showing changes by cost/capacity category
- Green bars indicate savings (positive values)
- Red bars indicate increases (negative values)
- Value labels in billions of CNY

### 2. Summary Table

A CSV summary table containing:

- Scenario code and description
- Total change (billion CNY)
- Top 3 categories with the largest changes and their amounts

### 3. Directory Structure

```
results/scenario_analysis/
├── scenario_plots/
│   └── scenario_comparison_costs.png
├── summary_tables/
│   └── scenario_summary_costs.csv
└── ...
```

### 4. Data Completeness Statistics

The script reports detailed data-completeness statistics, including:
- Total scenario count and completeness rate
- Data status for each scenario combination
- Types and counts of missing data

### File Format

- **costs.csv**: cost data with multi-level index (component_type, cost_type, carrier).
- **capacities.csv**: capacity data in a similar format.

## Cost Categories

The script organizes costs into the following categories:

1. **variable cost-non-renewable**: non-renewable variable cost
2. **capital-non-renewable**: non-renewable capital cost
3. **capital–demand side**: demand-side capital cost
4. **capital–renewable**: renewable capital cost
5. **transmission lines**: transmission lines
6. **batteries**: battery storage
7. **long-duration storages**: long-duration storage
8. **carbon capture**: carbon capture
9. **synthetic fuels**: synthetic fuels
10. **carbon management**: carbon management

## Troubleshooting

### Common Issues

1. **No scenario results found**
   - Check that the results directory exists.
   - Verify the version directory naming convention.
   - Confirm that summary files are complete.

2. **Data loading fails**
   - Check the CSV file format.
   - Confirm the file path is correct.
   - Inspect the detailed error log.

3. **Chart generation fails**
   - Check the matplotlib configuration.
   - Ensure sufficient memory is available.
   - Verify write permissions on the output directory.

### Debug Mode

Use the `--verbose` flag to enable verbose output, which lets you:

- See the data-loading status for each scenario.
- Monitor the cost categorization process.
- Inspect data transformations and calculations.

## Extension

### Custom Cost Categories

Modify the `cost_category_mapping` dictionary in the script to customize categories:

```python
cost_category_mapping = {
    ('marginal', 'coal'): 'Custom category name',
    # Add more mappings ...
}
```

### Adding New Analysis Dimensions

The script can be extended with additional analyses such as:

- Time-series analysis
- Geographic distribution analysis
- Technology-mix analysis

## Notes

1. **Memory**: processing many scenarios may require substantial memory.
2. **Runtime**: the first run may take longer while all data is loaded.
3. **Permissions**: make sure you have read access to the results directory and write access to the output directory.
4. **Data completeness**: it is recommended to verify that all scenario runs have finished before running the script.
